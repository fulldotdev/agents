# Week Planning Review

Use this for Sil's weekly business-coach planner. Keep it simple and practical.

## Default Notion Route

When Sil shares planner photos and asks to process/store them:

1. Read the handwriting from the planner photos.
2. Past week = previous Monday through Sunday.
3. Upcoming week = current Monday through Sunday when Sil is planning on Sunday/Monday.
4. Put `TERUGBLIK OP DE WEEK` in Notion `Reflects` page `Weekly YYYY-MM-DD` for the past week's Monday.
5. Put `FOCUS OP VOLGENDE WEEK` in Notion `Reflects` page `Weekly YYYY-MM-DD` for the upcoming week's Monday.
6. Create the `Reflects` weekly page if it does not exist.
7. Do not put planner content into client sprint tasks like Teveo/Fayn unless Sil explicitly names that client sprint.

## Data Sources

- Planner photos: wins, habits, learning, reflection question, personal notes, next-week focus, day planning.
- Notion Tasks database: progress from edited tasks in the past week.
- Goals database: yearly, long-term, and short-term goal context when needed.
- Progress means task status changes and relevant updates, especially tasks moved to `Waiting` or `Awaiting`.

## Workflow

1. OCR/read planner photos; mark unclear handwriting as uncertain instead of guessing.
2. Query Notion Tasks edited during the past week.
3. Summarize task progress:
   - completed work
   - moved-to-waiting/awaiting blockers
   - notable sales/growth/admin shifts
4. Write the past-week page with:
   - `TERUGBLIK OP DE WEEK`
   - 5 mooiste wins
   - goede gewoonte + streak
   - grootste learning
   - reflectievraag or answer from the planner
   - progressie uit Notion Tasks
   - vrije speelruimte when present
5. Write the upcoming-week page with:
   - `FOCUS OP VOLGENDE WEEK`
   - kiezen voor progressie
   - goede gewoonte
   - intentie
   - delen vanuit overvloed
   - professioneel/prive day planning
   - open handwriting uncertainties

## Rules

- The weekly reflection question changes every week; never hardcode it into the reusable template.
- Do not create, edit, cancel, or reopen tasks from planner content unless Sil explicitly asks.
- Notion weekly page writes are allowed when Sil asks to process planner photos into sprint/weekly pages.
- Prefer short Dutch output in Telegram.
- Ask a question only if the missing answer changes where content should be written.
